Asset (Group) management interfaces:
- ListAssetGroups: query the list of asset groups. 
- ListAssets: query the list of assets. 
- GetAsset: query asset information. 
- GetAssetGroup: query asset group information. 
- UpdateAssetGroup: update asset group information. 
- UpdateAsset: update asset information. 
 
Authentication method 
To call the Assets API interface, you must use Access Key authentication. For details, refer to Obtain API access keys (AK/SK). 
 
Rate limit requirement 
- QPS rate limit: The maximum number of API query requests per second. Query requests exceeding this limit will return an error.

CreateAssetGroup 
POST /open/CreateAssetGroup 
Create an Asset Group for asset management. 
When creating an Asset Group for the first time, you must sign an authorization letter in the console. For details, refer to [⚠️Confidential][Customer name required in application form] Private digital asset library (for invited users only).
Name string required
The name of the Asset Group, with a maximum of 64 characters. 

---
Description string
The description of the Asset Group, with a maximum of 300 characters. 

---
GroupType string
The type of the Asset Group. Optional values: 
- AIGC: Digital characters. 
Currently, only AIGC is supported.

---
ProjectName string
The name of the project to which the resource belongs. The default value is default. 
If the resource is not in the default project, you must enter the correct project name. For more information about project, see the related IAM docs.

---
Response parameters 
Id string
The Id of the Asset Group. 

---

CreateAsset 
POST /open/CreateAsset 
Create an Asset within the specified asset group. 
 
Request parameters 
GroupId string required
The Id of the Asset Group to which the asset belongs. 

---
URL string required
The publicly accessible address of the asset being uploaded. 

---
Name string
The name of the asset, up to 64 characters. 
This field is used only for fuzzy search when calling the ListAssets API and is not included in model inference. For details on generating videos with assets, see Generate videos using portrait assets and FAQ 4.

---
AssetType string required
Asset types support uploading images, audio, and video. Available values: 
- Image
- Video
- Audio
When uploading image, audio, or video assets, only URL upload is supported; base64 is not supported.
Requirements for a single image file
- Format: jpeg, png, webp, bmp, tiff, gif, heic/heif 
- Aspect ratio (width/height): (0.4, 2.5) 
- Width and height (px): (300, 6000) 
- Each image file must be less than 30 MB. 
Requirements for a single video
- Format: mp4, mov 
- Resolution: 480p, 720p 
- Each video must be between 2 and 15 seconds long. 
- Dimensions: 
  - Aspect ratio (width/height): [0.4, 2.5] 
  - Width and height (px): [300, 6000] 
  - Total pixel count: [640×640=409600, 834×1112=927408], that is, the product of width and height must fall within the range [409600, 927408]. 
- Size: each video must not exceed 50 MB 
- Frame rate (FPS): [24, 60] 
Requirements for a single audio
- Format: wav, mp3 
- Each audio must be between 2 and 15 seconds long. 
- Size: each audio must not exceed 15 MB 


---
ProjectName string
The name of the project to which the resource belongs. The default value is default. If the resource is not in the default project, you must enter the correct project name. For more information about project, see the related IAM docs. 
The ProjectName  must be consistent with the Asset Group to be created. 

Response parameters 
Id string
The Id of the asset (media asset). 
 
Return example: 
{
    "Id": "Asset-2026**********-*****"
}
 
ListAssets 
POST /open/ListAssets 
Query the list of assets (media assets) that meet the filter criteria. 
Request parameters 
Filter object required
Filter conditions for search. 

---
  Filter.GroupIds array
  The Id of the Asset Group to which the asset belongs. 

---
  Filter.GroupType string required
  The type of the asset group (media asset group). Optional values: 
  - AIGC: Digital characters. 

---
  Filter.Statuses array
  Task status. 
  - Active: The asset has been processed and is available for use. 
  - Processing: The asset is being pre-processed and is not available for use. 
  - Failed: The asset processing failed. 

---
Filter.Name string
The name of the asset, up to 64 characters. 

---
PageNumber int (i64) required
The search page number, which can be used for list pagination, starting from 1. For example: "page_number": 1, which returns the search results for the first page. 

---
PageSize int (i64) required
The number of search results per page, up to 100. 

---
SortBy string
The field name used for sorting. The default value is createTime. The following types are supported: 
- CreateTime: Sort by creation time. 
- UpdateTime: Sort by updated time. 
- GroupId: Sort by the Id of the asset group. 

---
SortOrder string
Sort order. The default value is Desc. Optional values: 
- Desc: Descending 
- Asc: Ascending 

---
ProjectName string
The name of the project to which the resource belongs. The default value is default. 
If the resource is not in the default project, you must enter the correct project name.  

---
 
Response parameters

An array of assets that meet the filter criteria. 

---

  Items.Id string
  The Id of the asset. 

---
  Items.name string
  The name of the asset, up to 64 characters. 

---
  Items.URL string
  The publicly accessible address of the asset. 

---
  Items.GroupId string
  The Id of the asset group to which the asset belongs. 

---
  Items.AssetType string
  The type of the asset. 
  - Image: The type of the asset is image. 

---
  Items.Status string
  Task status. 
  - Active: The asset has been processed and is available for use. 
  - Processing: The asset  is being pre-processed and is not available for use. 
  - Failed: The asset processing failed. 

---
  Items.Error object
  Error information. 

---
    Items.Error.Code string
    Error code. 

---
    Items.Error.Message string
    Error message. 

---
  Items.ProjectName string
  The name of the project to which the resource belongs.  

---
  Items.CreateTime string
  Creation time. 

---
  Items.UpdateTime string
  Updated time. 

---
TotalCount int (i64)
Return the total count. 

---
PageNumber int (i64)
Return the number of pages. 

---

The number of search results per page, up to 100.
